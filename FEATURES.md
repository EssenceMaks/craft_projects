# Component Generator v2 — Возможности, Правила и Функционал

## Архитектура

Одностраничное HTML-приложение с 5 вкладками. Всё состояние хранится в объекте `S` в памяти (без бэкенда, без localStorage).

### Глобальный объект `S`
```
S = {
  items[],         // Оригинальные элементы на холсте
  connections[],   // Анимационные связи (оригинал → копии)
  uiKit[],         // Атомы UI Kit
  sets[],          // Наборы элементов
  comps[],         // Сохранённые композиции
  selId,           // ID выбранного элемента
  selIds[],        // ID мультиселекта
  selIsCopy,       // true если выбрана копия
  selConnId        // ID связи выбранной копии
}
```

### Структура элемента (`S.items`)
```
{ id, name, group, html, css, js, parentId, copyOnlyConnIds[] }
```
- `parentId` — вложенность в DOM дереве
- `copyOnlyConnIds[]` — список connId, в которых элемент виден только при активации

### Структура связи (`S.connections`)
```
{
  id, label, trigger, triggerTargetId, animation, duration, easing,
  sourceIds[],   // ID оригиналов, которые были выделены
  fromConnId,    // ID родительской связи (для цепочек анимаций)
  copies: [{
    id, originalId, css, html, js,
    parentCopyId  // вложенность внутри копии (зеркало DOM оригинала)
  }]
}
```

## Вкладка 3 — Сборка (Холст + Анимации)

### Концепция (Penpot-стиль)
1. Разместить элементы на холсте из левой палитры
2. Выделить один или несколько (Shift+клик для мультиселекта)
3. Нажать **«Анимировать»** → создаётся копия с иерархией (см. ниже)
4. Стрелка-связь рисуется между оригиналами и копиями
5. Копии = конечное состояние анимации — их редактируют, двигают, изменяют

### Холст
- Виртуальный размер: 4000×3000px, прокрутка через скроллбары
- Элементы: `position: absolute` с `left`/`top`
- Перетаскивание обновляет CSS строку
- При выделении: синий контур + 8 ручек ресайза + тултип `Ш × В`
- Копии показывают метку «копия» (оранжевый бейдж) без пунктирного контура по умолчанию; пунктир появляется только при выделении

### Линейки (Rulers)
- Горизонтальная и вертикальная, тёмный фон
- Тики каждые 10px, подписи каждые 100px
- Синий крестик-курсор следует за мышью

### Иерархические копии (`parentCopyId`)
- При нажатии «Анимировать» создаётся **полное поддерево** оригиналов:
  - Каждый дочерний элемент оригинала получает соответствующую копию
  - Копии хранят `parentCopyId` — зеркало `parentId` оригиналов
- Копии рендерятся рекурсивно (функция `buildCopyHTML`)
- Добавление ребёнка к оригиналу → `syncItemToCopies` автоматически добавляет соответствующую копию

### Анимация от копии (цепочки анимаций)
- Можно выделить **копию** и нажать «Анимировать»
- Создаётся вторая связь (`fromConnId` указывает на первую)
- Начальное состояние = CSS первой копии, конечное = новая копия
- Стрелки рисуются от родительской копии, если задан `fromConnId`

### Позиционирование в анимации
- `left`/`top` из CSS копии **полосаются** в экспорте, если родитель элемента **не входил** в `sourceIds` при создании связи
- Правило: *анимировать элемент без родителя = менять только внутренние свойства, не позицию*
- Если нужна анимация позиции → включить родителя в выделение

### Левая панель
- Палитра из активного набора
- Кнопка **«Анимировать»** — активна при выделении 1+ элементов или копии

### Правая панель (Свойства)
- CSS, HTML, JS — редактируются для оригиналов **и копий**
- Изменения HTML/JS копии сохраняются в `cp.html`/`cp.js`
- **Блочная модель** — визуальный редактор W/H/Radius/Padding/Margin
  - Поля обновляют CSS textarea в реальном времени
  - При изменении CSS textarea — блочная модель синхронизируется
- **DOM Иерархия** (оригиналы) — drag-and-drop вложенность
- **Копии / Анимации** — дерево копий (см. ниже)

### DOM Иерархия — Оригиналы
- Drag карточки **на другую карточку** = вложить (изменить `parentId`)
- Drag **на синюю полосу** = переставить порядок
- Клик = выделить элемент на холсте

### DOM Иерархия — Копии (раздел «КОПИИ / АНИМАЦИИ»)
- Каждая связь отображается как сворачиваемая группа
- Иерархия копий зеркалит оригинальное дерево (отступы, вложенность)
- **Drag копии на другую копию** = вложить (изменить `parentCopyId`)
- **Drag копии на синюю полосу** = переставить / сменить родителя
- **Drag оригинала из дерева → drop на копию** = добавить элемент в копию:
  - Создаётся новая копия с `parentCopyId` = целевая копия
  - Оригинал помечается `copyOnlyConnIds: [connId]`
  - Оригинал скрыт по умолчанию (`display: none`)
  - Виден только когда данная связь активна (`display: block`)
- Клик по копии в DOM = выделить её на холсте

### Нижняя панель (Предпросмотр)
- iframe с живым превью включая HTML/JS изменений копий
- Кнопка «Обновить», кнопка «Скрыть/Показать»
- **Перетаскиваемый верхний край** — изменение высоты панели

### Стрелки-связи
- Bezier кривая от оригиналов к корневым копиям
- Если `fromConnId` задан — стрелка рисуется от родительской копии
- Бейдж посередине: клик → модалка настройки анимации

## Вкладка 4 — Экспорт и Предпросмотр

- Полное интерактивное превью
- **«Скачать HTML»** — Web Component
- **«Сохранить»** → в Галерею + в Композиции

### Правила экспорта (`buildExport`)
- Web Component (`<custom-smart-card>`) с Shadow DOM
- `defaultCss`: позиции всех оригиналов + `transition: all var(--dur) var(--ease)`
  - Элементы с `copyOnlyConnIds` получают `display: none` по умолчанию
- `connCss`: при активной связи (`act_connId`) — CSS копии переопределяет оригинал
  - `left`/`top` исключены, если родитель не в `sourceIds`
  - `display: block` добавляется для `copyOnlyConnIds` элементов
- `connJs`: триггеры hover/click + HTML swaps (если `cp.html ≠ orig.html`)
  - HTML swap через `${tid}swap(true/false)` при активации/деактивации
- `customJs`: пользовательский JS из `it.js` (с заменой `{{ID}}`)
- Поддержка `tog()` с View Transitions API (`vt` анимация)
- `previewHTML`: упрощённая версия без Shadow DOM для нижней панели

## View Transitions API — пояснение

**View Transitions API** — это браузерный механизм плавных переходов между двумя визуальными состояниями.

**Как работает**:
1. Браузер делает «снимок» текущего состояния страницы
2. Применяются изменения DOM (CSS классы, перестройка грид-сетки)
3. Браузер анимирует переход от снимка к новому состоянию

**Зачем нужен в нашем контексте**:
- При использовании `position: absolute` анимацию нужно вручную задавать через `left`/`top`
- При использовании Grid/Flexbox элементы сами расставляются по сетке — не нужно задавать координаты
- View Transitions API анимирует **перемещение элементов из одной позиции сетки в другую автоматически**
- Каждому анимируемому элементу назначается `view-transition-name`, браузер отслеживает его перемещение

**Практический вывод**: VT API позволяет создавать плавные анимации в системах с grid/flex-лейаутом, не прибегая к абсолютному позиционированию. Элемент «знает», где он должен быть в каждом состоянии — Layout Engine считает это за вас.

## Глобальные утилиты

- `setCss(css, prop, val)` — устанавливает свойство в CSS строку через DOM (корректный парсинг)
- `getV(css, prop)` — regex-поиск свойства в CSS строке
- `parsePropVal(css, prop)` — как `getV`, но дополнительно разворачивает шортхенды (`padding` → `padding-top` и т.д.)
- `prevCss(css)` — убирает `position:absolute`, `left:`, `top:` для превью
- `uid()` — 6-символьный random ID
- `lucide.createIcons()` — вызывается после каждой мутации innerHTML
