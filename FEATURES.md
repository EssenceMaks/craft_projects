# Component Generator v2 — Возможности, Правила и Функционал

## Архитектура

Одностраничное HTML-приложение с 5 вкладками. Всё состояние хранится в объекте `S` в памяти (без бэкенда, без localStorage).

### Глобальный объект `S`
```
S = {
  items[],         // Оригинальные элементы на холсте
  connections[],   // Анимационные связи (оригинал → копии)
  uiKit[],         // Атомы UI Kit
  sets[],          // Наборы элементов
  comps[],         // Сохранённые композиции
  selId,           // ID выбранного элемента
  selIds[],        // ID мультиселекта
  selIsCopy,       // true если выбрана копия
  selConnId        // ID связи выбранной копии
}
```

### Структура элемента (`S.items`)
```
{ id, name, group, html, css, js, parentId, copyOnlyConnIds[] }
```
- `parentId` — вложенность в DOM дереве
- `copyOnlyConnIds[]` — список connId, в которых элемент виден только при активации

### Структура связи (`S.connections`)
```
{
  id, label, trigger, triggerTargetId, animation, duration, easing,
  sourceIds[],   // ID оригиналов, которые были выделены
  fromConnId,    // ID родительской связи (для цепочек анимаций)
  copies: [{
    id, originalId, css, html, js,
    parentCopyId  // вложенность внутри копии (зеркало DOM оригинала)
  }]
}
```

## Вкладка 3 — Сборка (Холст + Анимации)

### Концепция (Penpot-стиль)
1. Разместить элементы на холсте из левой палитры
2. Выделить один или несколько (Shift+клик для мультиселекта)
3. Нажать **«Анимировать»** → создаётся копия с иерархией (см. ниже)
4. Стрелка-связь рисуется между оригиналами и копиями
5. Копии = конечное состояние анимации — их редактируют, двигают, изменяют

### Холст
- Виртуальный размер: 4000×3000px, прокрутка через скроллбары
- Элементы: `position: absolute` с `left`/`top`
- Перетаскивание обновляет CSS строку
- При выделении: синий контур + 8 ручек ресайза + тултип `Ш × В`
- Копии показывают метку «копия» (оранжевый бейдж) без пунктирного контура по умолчанию; пунктир появляется только при выделении

### Линейки (Rulers)
- Горизонтальная и вертикальная, тёмный фон
- Тики каждые 10px, подписи каждые 100px
- Синий крестик-курсор следует за мышью

### Иерархические копии (`parentCopyId`)
- При нажатии «Анимировать» создаётся **полное поддерево** оригиналов:
  - Каждый дочерний элемент оригинала получает соответствующую копию
  - Копии хранят `parentCopyId` — зеркало `parentId` оригиналов
- Копии рендерятся рекурсивно (функция `buildCopyHTML`)
- Добавление ребёнка к оригиналу → `syncItemToCopies` автоматически добавляет соответствующую копию

### Анимация от копии (цепочки анимаций)
- Можно выделить **копию** и нажать «Анимировать»
- Создаётся вторая связь (`fromConnId` указывает на первую)
- Начальное состояние = CSS первой копии, конечное = новая копия
- Стрелки рисуются от родительской копии, если задан `fromConnId`

### Позиционирование в анимации
- `left`/`top` из CSS копии **полосаются** в экспорте, если родитель элемента **не входил** в `sourceIds` при создании связи
- Правило: *анимировать элемент без родителя = менять только внутренние свойства, не позицию*
- Если нужна анимация позиции → включить родителя в выделение

### Левая панель
- Палитра из активного набора
- Кнопка **«Анимировать»** — активна при выделении 1+ элементов или копии

### Правая панель (Свойства)
- CSS, HTML, JS — редактируются для оригиналов **и копий**
- Изменения HTML/JS копии сохраняются в `cp.html`/`cp.js`
- **Блочная модель** — визуальный редактор W/H/Radius/Padding/Margin
  - Поля обновляют CSS textarea в реальном времени
  - При изменении CSS textarea — блочная модель синхронизируется
- **DOM Иерархия** (оригиналы) — drag-and-drop вложенность
- **Копии / Анимации** — дерево копий (см. ниже)

### DOM Иерархия — Оригиналы
- Drag карточки **на другую карточку** = вложить (изменить `parentId`)
- Drag **на синюю полосу** = переставить порядок
- Клик = выделить элемент на холсте

### DOM Иерархия — Копии (раздел «КОПИИ / АНИМАЦИИ»)
- Каждая связь отображается как сворачиваемая группа
- Иерархия копий зеркалит оригинальное дерево (отступы, вложенность)
- **Drag копии на другую копию** = вложить (изменить `parentCopyId`)
- **Drag копии на синюю полосу** = переставить / сменить родителя
- **Drag оригинала из дерева → drop на копию** = добавить элемент в копию:
  - Создаётся новая копия с `parentCopyId` = целевая копия
  - Оригинал помечается `copyOnlyConnIds: [connId]`
  - Оригинал скрыт по умолчанию (`display: none`)
  - Виден только когда данная связь активна (`display: block`)
- Клик по копии в DOM = выделить её на холсте

### Нижняя панель (Предпросмотр)
- iframe с живым превью включая HTML/JS изменений копий
- Кнопка «Обновить», кнопка «Скрыть/Показать»
- **Перетаскиваемый верхний край** — изменение высоты панели

### Стрелки-связи
- Bezier кривая от оригиналов к корневым копиям
- Если `fromConnId` задан — стрелка рисуется от родительской копии
- Бейдж посередине: клик → модалка настройки анимации

## Вкладка 4 — Экспорт и Предпросмотр

- Полное интерактивное превью
- **«Скачать HTML»** — Web Component
- **«Сохранить»** → в Галерею + в Композиции

### Правила экспорта (`buildExport`)
- Web Component (`<custom-smart-card>`) с Shadow DOM
- `defaultCss`: позиции всех оригиналов + `transition: all var(--dur) var(--ease)`
  - Элементы с `copyOnlyConnIds` получают `display: none` по умолчанию
- `connCss`: при активной связи (`act_connId`) — CSS копии переопределяет оригинал
  - `left`/`top` исключены, если родитель не в `sourceIds`
  - `display: block` добавляется для `copyOnlyConnIds` элементов
- `connJs`: триггеры hover/click + HTML swaps (если `cp.html ≠ orig.html`)
  - HTML swap через `${tid}swap(true/false)` при активации/деактивации
- `customJs`: пользовательский JS из `it.js` (с заменой `{{ID}}`)
- Поддержка `tog()` с View Transitions API (`vt` анимация)
- `previewHTML`: упрощённая версия без Shadow DOM для нижней панели

## View Transitions API — пояснение

**View Transitions API** — это браузерный механизм плавных переходов между двумя визуальными состояниями.

**Как работает**:
1. Браузер делает «снимок» текущего состояния страницы
2. Применяются изменения DOM (CSS классы, перестройка грид-сетки)
3. Браузер анимирует переход от снимка к новому состоянию

**Зачем нужен в нашем контексте**:
- При использовании `position: absolute` анимацию нужно вручную задавать через `left`/`top`
- При использовании Grid/Flexbox элементы сами расставляются по сетке — не нужно задавать координаты
- View Transitions API анимирует **перемещение элементов из одной позиции сетки в другую автоматически**
- Каждому анимируемому элементу назначается `view-transition-name`, браузер отслеживает его перемещение

**Практический вывод**: VT API позволяет создавать плавные анимации в системах с grid/flex-лейаутом, не прибегая к абсолютному позиционированию. Элемент «знает», где он должен быть в каждом состоянии — Layout Engine считает это за вас.

---

## Глобальные утилиты

- `setCss(css, prop, val)` — устанавливает свойство в CSS строку через DOM (корректный парсинг)
- `getV(css, prop)` — regex-поиск свойства в CSS строке
- `parsePropVal(css, prop)` — как `getV`, но дополнительно разворачивает шортхенды (`padding` → `padding-top` и т.д.)
- `prevCss(css)` — убирает `position:absolute`, `left:`, `top:` для превью
- `uid()` — 6-символьный random ID
- `lucide.createIcons()` — вызывается после каждой мутации innerHTML

# Neon Bubbles Pro — Возможности и Функционал

## Архитектура

PixiJS 7 WebGL GPU-accelerated бесконечный холст. Всё состояние хранится в `window.state`. Файл проекта — JSON с полным снапшотом.

### Глобальный объект `state`
```
state = {
  bubbles: {},       // Главные баблы { id, name, x, y, size, shape, bgColor, borderColor, glowColor }
  minis:   {},       // Мини-баблы { id, name, parentId, x, y, bgColor, borderColor, glowColor }
  links:   {},       // Линии { id, from, to, waypoints[], type, lineMode, gap, labels[],
                     //         color1/2, width1/2, animType1/2, speed1/2, hasGlow, hasBg, useGlobalAnim, ... }
  points:  {},       // Якорные точки { x, y, attachedTo, angle, _renderedX, _renderedY }
  globalAnimConfig:  // { mode, shape, emojis, size, count, wobble, hideLines, ecoMode, hasGlow, speed }
  animationMode:     // 'play' | 'pause' | 'solid'
}
```

### Файл снапшота (JSON)
```json
{
  "state": { ...state },
  "camera": { "x", "y", "scale" },
  "minimap": { "visible", "state", "minimized" }
}
```

---

## Сущности холста

### Главный Бабл (`main_*`)
- Круг или квадрат, задаётся полем `shape`
- Настройки: имя, размер (100–600), цвет фона (rgba), цвет контура, цвет свечения
- Гло-эффект (свечение вокруг)
- Иконка ⚙️ для открытия свойств
- Перетаскивание с коллизийным расталкиванием других баблов
- Мультиселект (lasso): drag по пустому месту = рамка выделения; перетаскивание любого из выделенных двигает всю группу

### Мини-Бабл (`mini_*`)
- Маленький прямоугольный лейбл внутри или снаружи главного бабла
- `parentId` — к какому баблу прикреплён (координаты относительные)
- При перетаскивании отпускание над другим баблом = смена `parentId`
- При отпускании в пустоте — `parentId = null`, абсолютные координаты
- Собственные цвета (фон, контур, свечение)

### Линии / Связи (`link_*`)
- Соединяют **точки** (`point_*`), которые могут быть:
  - Свободными (x,y в мировом пространстве)
  - Прикреплёнными к центру бабла (`attachedTo + angle=null`)
  - Прикреплёнными к контуру бабла под углом (`attachedTo + angle`)
- Промежуточные точки — **вейпоинты** (waypoints)
- Типы траектории: `curved` (сплайн), `sharp` (ломаная), `ortho`, `bent` (Bent Connector с ручками-слайдерами)
- Режимы: `single` / `double` (две параллельные линии с зазором)
- Фоновая подложка (полоса за линией, настраиваемая)
- **GPU анимация Слоя 1 и Слоя 2**:
  - `pixi_dash_fwd / bwd` — движущиеся штрихи (вперёд/назад)
  - `pixi_dots_fwd / bwd` — движущиеся точки
  - `pixi_shapes` — частицы-фигуры (капля / куб / гексагон / круг)
  - `pixi_energy` — энергетические сгустки
  - `pixi_symbols` — эмодзи-частицы (настраиваемый набор)
  - `none` — статичная линия
- Скорость: абсолютная (пикс/сек) или относительная (сек/длину)
- Реверс анимации
- Индивидуальный glow с настраиваемой яркостью
- **Надписи на линии** (`labels[]`):
  - Тип: вдоль линии (`along`) или выноска (`callout`)
  - Позиция (0–100% длины)
  - Редактирование текста inline

### Точки (`p_*`)
- Видны при выделении линии или в режиме соединения
- Цвет: зелёный (1 линия) / оранжевый (≥2 линии)
- ПКМ → контекстное меню: связь отсюда, блок здесь, подписать, свойства линии, удалить точку
- Перетаскивание: snap к центру/контуру/другой точке бабла (радиус 30 worldpx)
- Слияние: перетащить на другую точку = объединить (все линии перецепляются)
- Разделение точки (split): одна точка → несколько независимых

## Камера и навигация

- **Скролл**: зум к курсору (ZOOM_MIN=0.05, ZOOM_MAX=5.0)
- **ПКМ/ПКМ+drag**: перемещение камеры
- **Пробел + ЛКМ**: перемещение камеры
- **WASD**: панорамирование (15px/кадр)
- **Краевое панорамирование**: курсор к краю (30px) — автоматическое движение
- **Ограниченная камера** (`isBounded=true`): камера не выходит за пределы сцены, кнопки `+` расширяют границы
- **Свободная камера** (`isBounded=false`): без ограничений
- `applyCameraBounds()` — принудительно корректирует позицию и масштаб

## Инструменты и режимы работы

### Режим создания (Create Dropdown)
- **Создать Бабл** — ЛКМ на холсте или кнопка тулбара
- **Режим Соединения** (link) — клик = начало, ещё клик = конец; Escape = отмена
- **Режим Линии** (line) — последовательные клики = вейпоинты, ПКМ = завершить
- В разработке: Тикет, Текст, Картинка

### Контекстные меню (ПКМ)
- На пустом холсте: создать бабл, создать связь, создать линию
- На бабле: создать мини-бабл, свойства, удалить
- На мини-бабле: свойства, удалить
- На линии: ➕ блок здесь, 🔗 связь отсюда, 📝 подписать, ⚙️ свойства, (сброс изгиба для bent)
- На точке: разделить, связь отсюда, блок здесь, подписать, свойства линии, удалить

### Bent Connector
- Ручки-слайдеры на каждом сегменте для перетаскивания
- Авто-слияние параллельных сегментов (< 15px)
- Очистка линейных вейпоинтов (`cleanupBentWaypoints`)

- Блок 2: Global Pattern Dropdown

## Миникарта

- Canvas 2D поверх PixiJS канваса

- Перетаскиваемый HUD (drag handle или Shift+клик на канвасе)
- Сворачивание (кнопка `_`), закрытие (❌), resize (🔳)
- Клик/drag по миникарте = навигация камеры к точке
- Отображает: баблы (цветные пятна), текущий viewport (белый прямоугольник)
- Хоткей **M** — toggle (small → max → off → small)

## Панель Свойств

- Плавающая, перетаскиваемая (drag за заголовок)

  - Действия: Копировать, Удалить

## Рендер цикл (PixiJS Ticker)

- `queueRender()` — устанавливает `needsRender=true`, запускает ticker если стоял
- При отсутствии анимации и движения — `ticker.stop()` (0% CPU в покое)
- Слои отрисовки (в порядке Z):
  1. `layerBg` — фоновые подложки линий
  2. `layerGlow` — свечение линий
  3. `layerLines` — сами линии (l1, l2)
  4. `layerPart` — частицы на линиях (non-interactive)
  5. `layerLbl` — надписи на линиях (non-interactive)
  6. `layerHit` — hit-зоны линий + lasso-прямоугольник
  7. `layerBub` — баблы и мини-баблы
  8. `layerPts` — точки

### Оптимизации
- `cheapHash`: если геометрия/стиль линии не изменились — не пересчитывать `getAllPts` и LUT
- Dash-анимация: перерисовывается только каждые 3 кадра (`dashTick % 3`)
- Particle система: pooling не применяется, но dead-particles удаляются
- ЭКО-режим: отключает `backdrop-filter` глобально

# АНАЛИЗ СЛИЯНИЯ: Workspace = Bubble + Component Generator

## Концепция

| Аспект | Bubble | Component Generator | Merged Workspace |
|--------|--------|---------------------|-----------------|
| Холст | PixiJS GPU, бесконечный | HTML/CSS, 4000×3000 | **PixiJS** главный |
| Сущности | Баблы + мини + линии | items + copies + connections | Баблы содержат **CG-рабочее пространство** |
| Сохранение | JSON файл | Supabase + localStorage | **Supabase** primary, JSON экспорт |
| Авторизация | PIN 6 цифр | Пароль (TEAM_USERS) | **TEAM_USERS** с паролем (унифицировано) |
| Коллаборация | Отсутствует | Supabase Realtime | **Полная**: global-channel, live sessions, курсоры |
| Контекст-бар | Отсутствует | cloud-bar в шапке | **cloud-bar** в шапке Workspace |

## Структура данных Workspace

```js
window.state = {
  // Bubble сущности (оригинал)
  bubbles: {},
  minis:   {},
  links:   {},
  points:  {},
  globalAnimConfig: {},
  animationMode: 'play',

  // CG данные — по одному объекту на каждый "хост-бабл"
  cgData: {
    [bubbleId]: {
      items:       [],  // S.items
      connections: [],  // S.connections
      uiKit:       [],  // атомы
      sets:        [],  // наборы
      comps:       [],  // сохранённые композиции
    }
  }
}
```

## CG Tab Mini-Bubbles

Правый клик на бабл → **"🧩 Создать окна компонент генератора"**:
1. Создаётся `state.cgData[bubbleId]` с дефолтными атомами и пустыми структурами
2. Внутри бабла создаются **5 мини-баблов** типа `cg_tab`:
   - `mini_cg_1` — **UI Kit** (левая верхняя позиция)
   - `mini_cg_2` — **Наборы** (правая верхняя)
   - `mini_cg_3` — **Сборка** (центр, самый большой)
   - `mini_cg_4` — **Экспорт** (левая нижняя)
   - `mini_cg_5` — **Галерея** (правая нижняя)
3. Мини-баблы отображаются как **квадратные** плашки с иконкой вкладки
4. Клик по CG-мини-баблу → открывает HTML-панель-оверлей с контентом вкладки

## Панели CG (HTML Overlay)

- Фиксированная `position:fixed` панель, переключается при смене активного мини-бабла
- Каждая панель содержит полный оригинальный UI соответствующей вкладки CG
- При изменении данных в панели → `state.cgData[bubbleId]` обновляется → `broadcastCGUpdate()` (Supabase)
- Панели можно перемещать независимо
- Размеры: Tab 3 (Сборка) — большой, остальные — компактнее

## Supabase — схема хранения

Объект сохранения в `projects` таблице:
```json
{
  "id": "ws_project1_экземпляр_3",
  "data": {
    "bubbles": {},
    "minis": {},
    "links": {},
    "points": {},
    "globalAnimConfig": {},
    "animationMode": "play",
    "cgData": { "bubbleId": { "items":[], "connections":[], ... } }
  },
  "camera": { "x", "y", "scale" },
  "minimap": {}
}
```

## Live Sessions (Workspace)

- Каждый пользователь при старте LIVE: upsert `live_{userId}` в `projects`
- Global channel `cg_global`: go_live / go_offline broadcasts → тост + зелёная точка на кнопке Проекты
- Broadcast события:
  - `canvas_update` — изменение bubble/mini/link/points
  - `cg_update` — изменение cgData[bubbleId]
  - `cursor` — позиция курсора (мировые координаты)
  - `note_update` — заметка e+Enter
- Watch mode: readonly (наблюдатель) или edit (со-редактор)

## Горячие клавиши (объединённые)

| Клавиша | Действие |
|---------|----------|
| WASD | Панорама камеры |
| M / М | Toggle миникарта |
| Ctrl+Z | Undo |
| Ctrl+Y / Ctrl+Shift+Z | Redo |
| Escape | Отмена режима создания |
| E + Enter | Создать/убрать заметку |
| Delete | Удалить выбранную сущность |

## Правила хранения CG состояния

- CG-холст Tab 3 — собственная `renderCanvas()` функция, работает внутри панели (iframe-like div)
- Активный CG-контекст: глобальная переменная `activeCgCtx = { bubbleId, S }`
- Переключение: при клике другого CG-мини — сохраняем текущий S → загружаем новый → ререндер
- При live broadcast `cg_update` — если получатель имеет открытую панель того же bubbleId — rerenderCanvas
